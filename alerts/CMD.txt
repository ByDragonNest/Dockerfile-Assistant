[+] BEST PRACTICE: You can use the exec form of ENTRYPOINT to set fairly stable default commands and arguments and then use either form of CMD to set additional defaults that are more likely to be changed.
[+] BEST PRACTICE: If you are not familiar with ENTRYPOINT, then always use CMD in the form: CMD [\"executable\", \"param1\", \"param2\"].
[?] REMINDER: Don't confuse RUN with CMD. RUN actually runs a command and commits the result; CMD doesn't execute anything at build time, but specifies the intended command for the image.
[!] WARNING: If you add the ENTRYPOINT instruction, it will override all elements specified using CMD.
